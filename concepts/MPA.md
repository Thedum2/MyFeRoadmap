# MPA

#개념

MPA는 페이지 이동마다 서버가 새 HTML 문서를 반환하는 전통적 웹 구조다. 문서 중심 웹의 기본 모델로, 요청 단위 경계가 명확하다.

장점은 단순성이다. 서버가 렌더링 결과를 책임지므로 초기 표시와 SEO 관점에서 유리하고, 페이지 단위 캐시 전략도 직관적이다.

low-level에서는 링크 클릭마다 HTTP 요청이 발생하고, 브라우저는 새 문서를 파싱해 렌더링 파이프라인을 다시 수행한다. 전환 비용은 있지만 책임 분리가 명확하다.

한계는 빈번한 페이지 전환 시 체감 속도 저하와 중복 로딩이다. 그래서 부분 갱신 기술이나 SSR 최적화, 하이브리드 구조가 많이 채택된다.
