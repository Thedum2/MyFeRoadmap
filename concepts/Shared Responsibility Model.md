# Shared Responsibility Model

#개념

Shared Responsibility Model은 클라우드 보안·운영 책임을 공급자와 사용자 사이에 분담하는 원칙이다. 무엇을 누가 책임지는지 경계를 명확히 하는 모델이다.

클라우드는 모든 책임을 대신해주지 않는다. 인프라 하부는 공급자가 책임지더라도, 계정 권한, 데이터 보호, 애플리케이션 설정은 사용자가 책임지는 경우가 많다.

low-level에서 이 모델은 IAM 정책, 네트워크 접근 제어, 암호화 키 관리, 로깅 감사 설계로 구체화된다. 경계 해석이 틀리면 보안 공백이 생긴다.

서비스 모델이 올라갈수록 사용자 책임 범위는 줄지만, 완전히 사라지지는 않는다. 그래서 IaaS/PaaS/SaaS별 책임표를 운영 기준으로 유지하는 것이 중요하다.
