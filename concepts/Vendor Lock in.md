# Vendor Lock in

#개념

Vendor Lock in은 특정 클라우드나 플랫폼의 고유 기능 의존이 커져 다른 환경으로 이전하기 어려워지는 상태다. 기술적·운영적 전환 비용이 급격히 높아진다.

문제의 핵심은 단순 사용 여부가 아니라 의존 깊이다. 데이터 구조, 관리형 서비스 API, 운영 절차가 특정 벤더 방식에 묶일수록 탈출 비용이 커진다.

low-level에서는 데이터 마이그레이션, 네트워크 재구성, 인증 체계 이전, 운영 자동화 재작성 비용이 주요 부담으로 나타난다.

완화 전략은 추상화와 표준화다. 멀티클라우드 자체가 목표가 아니라, 핵심 경로에서 교체 가능성을 유지하는 설계가 목표가 된다.
