# 웹 브라우저

#개념

웹 브라우저는 서버가 보낸 웹 자원을 해석해 사용자 화면으로 바꾸는 실행 환경이다. 단순한 뷰어가 아니라 HTML, CSS, JavaScript를 파싱하고 실행하는 플랫폼이다.

초기 브라우저는 문서 열람에 집중했다. 링크를 클릭해 새 문서를 받아 보여주는 구조였다. 이후 상호작용 요구가 커지면서 스크립트 실행, 보안 경계, 성능 최적화가 브라우저의 핵심 책임이 됐다.

low-level에서는 네트워크 스택, 파서, 렌더링 엔진, JavaScript 엔진이 파이프라인으로 연결된다. URL 요청으로 받은 자원을 파싱해 DOM/CSSOM을 만들고, 레이아웃 계산과 페인트를 거쳐 픽셀을 그린다.

브라우저의 한계는 안전성과 성능의 균형이다. 외부 코드를 실행해야 하면서도 시스템을 보호해야 하므로 샌드박스, 동일 출처 정책, 권한 모델이 중요해진다.
