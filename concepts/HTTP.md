# HTTP

#개념

HTTP는 웹에서 클라이언트와 서버가 자원을 주고받기 위한 애플리케이션 계층 프로토콜이다. 웹 요청과 응답의 문법과 의미를 표준화한다.

핵심 설계는 요청-응답 모델이다. 클라이언트가 메서드와 경로, 헤더를 담아 요청하면 서버가 상태 코드와 본문으로 응답한다. 이 단순 모델이 웹 생태계 전체의 공통 기반이 됐다.

low-level에서는 TCP/IP 같은 전송 계층 위에서 메시지가 전달된다. 캐시 제어, 압축, 콘텐츠 협상, 인증 같은 기능은 헤더 규약을 통해 확장된다.

한계는 상태 비보존과 헤더 오버헤드에서 자주 드러난다. 그래서 쿠키, 세션, 토큰 전략과 HTTP 2/3 같은 성능 개선 계열이 발전했다.
