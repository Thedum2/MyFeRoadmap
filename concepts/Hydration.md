# Hydration

#개념

Hydration은 서버가 먼저 그려준 HTML에 클라이언트 JavaScript 이벤트와 상태를 연결해 상호작용 가능한 UI로 전환하는 과정이다.

SSR만으로는 정적 화면 표시까지만 가능하다. 사용자가 클릭하고 입력하려면 브라우저 측 로직이 DOM과 다시 결합돼야 한다. 이 결합 단계가 하이드레이션이다.

low-level에서는 클라이언트 런타임이 기존 DOM을 재사용하면서 이벤트 핸들러를 바인딩하고 내부 상태를 동기화한다. 완전 재렌더 대신 재사용을 목표로 성능을 확보한다.

한계는 하이드레이션 비용이 큰 페이지에서 지연이 느껴질 수 있다는 점이다. 그래서 부분 하이드레이션, 아일랜드 아키텍처 같은 전략이 발전했다.
