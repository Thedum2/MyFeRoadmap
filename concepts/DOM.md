# DOM

#개념

DOM은 브라우저가 HTML 문서를 메모리에서 다루기 위해 만든 객체 트리다. 페이지를 코드로 조작할 수 있는 이유가 DOM 구조 덕분이다.

문서를 단순 문자열로 유지하면 부분 수정이 어렵다. DOM은 문서 요소를 노드 객체로 분해해 부모-자식 관계를 갖는 트리로 만든다. 그래서 특정 노드 탐색, 수정, 삭제가 가능해진다.

low-level에서는 파서가 HTML을 읽어 노드를 만들고 트리에 배치한다. JavaScript는 DOM API로 노드를 조작하고, 변경이 생기면 렌더링 파이프라인이 다시 계산된다.

DOM 조작이 과도하면 레이아웃/페인트 비용이 커질 수 있다. 그래서 변경을 묶거나 가상 DOM 같은 추상화를 활용해 비용을 줄이는 전략이 등장했다.
