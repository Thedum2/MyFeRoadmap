# Code Splitting

#개념

Code Splitting은 JavaScript 번들을 기능 단위로 쪼개 필요한 시점에만 로드하는 최적화 기법이다. 초기 로딩 부담을 줄이는 데 목적이 있다.

SPA가 커질수록 첫 진입 시 내려받아야 할 코드가 급격히 늘어난다. 코드를 경로/컴포넌트 단위로 분할하면 첫 화면 표시 시간을 줄일 수 있다.

low-level에서는 번들러가 의존 그래프를 분석해 청크를 만들고, 런타임이 동적 import 시점에 청크를 비동기로 요청한다. 캐시와 조합하면 재방문 성능이 개선된다.

한계는 청크가 너무 잘게 쪼개질 때 요청 수 증가와 로딩 지연이 생길 수 있다는 점이다. 분할 경계를 제품 사용 흐름에 맞춰 설계해야 한다.
