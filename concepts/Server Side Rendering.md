# Server Side Rendering

#개념

Server Side Rendering은 서버가 요청 시점에 HTML을 만들어 응답하는 렌더링 방식이다. 클라이언트는 완성된 마크업을 먼저 받아 빠르게 첫 화면을 볼 수 있다.

CSR의 초기 표시 지연과 SEO 한계를 보완하기 위해 SSR이 널리 쓰인다. 특히 콘텐츠 탐색, 검색 유입, 소셜 미리보기에서 효과가 크다.

low-level에서는 서버가 템플릿/컴포넌트를 실행해 HTML을 생성하고 전송한다. 이후 클라이언트 JavaScript가 붙어 상호작용을 활성화하는 과정이 하이드레이션이다.

한계는 서버 연산 비용과 캐시 전략 복잡성이다. 요청당 렌더링 부담이 커지므로 스트리밍, 캐시 계층, 점진적 렌더링 설계가 중요하다.
